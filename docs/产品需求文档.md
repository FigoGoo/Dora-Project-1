# Dora 魔盒产品需求文档

## 一、产品概述

### 1.1 产品定位
Dora 魔盒是一个基于 AI 的全流程视频创作 SaaS 平台，用户只需输入灵感，即可自动完成从剧本生成到视频发布的全链路工作。

### 1.2 核心价值
- **降低门槛**：无需专业视频制作技能，人人皆可创作视频
- **提升效率**：AI 驱动的自动化流程，大幅缩短创作周期
- **灵活配置**：支持自定义 AI 模型，满足不同创作需求

### 1.3 目标用户
- 内容创作者（自媒体、UP主）
- 电商运营人员（商品展示视频）
- 教育工作者（教学视频制作）
- 营销人员（广告宣传片）

## 二、核心功能模块

### 2.1 项目管理模块
| 功能 | 说明 |
|------|------|
| 项目创建 | 新建视频项目，填写基本信息（标题、描述、标签） |
| 项目列表 | 展示用户所有项目，支持搜索、筛选、排序 |

| 项目复制 | 基于已有项目创建新项目 |

|------|------|
| 灵感输入 | 支持文本输入、关键词列表、参考链接等多种形式 |

| 版本管理 | 保存剧本历史版本，支持对比和回滚 |

| 镜头提示词 | 为每个镜头生成 AI 绘图提示词 |

## 四、业务流程详细说明

### 4.1 核心创作流程

```
灵感输入 → 剧本生成 → 分镜拆解 → 画面绘制 → 视频生成 → 视频拼接 → 视频下载/发布
```

### 4.2 详细流程说明

#### 4.2.1 项目创建
1. 用户点击「新建项目」
2. 填写项目基本信息（标题、描述）
3. 选择项目模板（可选）
4. 系统创建项目并进入创作流程

#### 4.2.2 灵感输入与剧本生成
1. 用户在剧本编辑器中输入灵感
   - 方式一：自由文本描述
   - 方式二：关键词列表
   - 方式三：上传参考文章/视频链接
2. 选择剧本风格和目标时长
3. 选择使用的文本模型（DeepSeek / Gemini）
4. 点击「生成剧本」
5. 系统调用 AI 模型生成剧本
6. 用户在编辑器中查看和修改剧本
7. 满意后进入下一步

#### 4.2.3 分镜拆解
1. 系统自动将剧本拆分为多个镜头
2. 生成分镜列表，包含：
   - 镜头序号
   - 时长
   - 画面描述
   - 台词/旁白
   - 运镜方式
   - AI 绘图提示词
3. 用户在时间轴上查看和调整分镜
4. 可手动修改每个镜头的提示词
5. 确认后进入画面绘制

#### 4.2.4 画面绘制
1. 用户选择图片模型（Banana 2）
2. 设置画面风格、分辨率等参数
3. 可选择上传参考图统一画风
4. 点击「批量生成」
5. 系统为每个镜头生成 2-4 张候选图
6. 用户为每个镜头选择最佳画面
7. 不满意的镜头可重新生成
8. 确认所有画面后进入视频生成

#### 4.2.5 视频生成
1. 用户选择视频模型（Seedance）
2. 为每个镜头设置运动方式：
   - 固定镜头
   - 向左/右平移
   - 向上/下平移
   - 推近/拉远
   - 旋转
3. 设置视频参数（帧率、质量）
4. 点击「生成视频」
5. 系统逐个生成视频片段
6. 用户预览每个片段
7. 不满意可重新生成
8. 确认后进入视频拼接

#### 4.2.6 视频拼接
1. 系统自动将所有片段按顺序排列在时间轴上
2. 用户可调整片段顺序和时长
3. 添加转场效果
4. 添加音频：
   - 背景音乐（从曲库选择或上传）
   - AI 配音（选择音色、语速）
   - 音效
5. 生成字幕，支持编辑和样式调整
6. 添加水印（可选）
7. 预览完整视频
8. 确认后进入导出

#### 4.2.7 视频下载与发布
**下载流程：**
1. 选择输出格式（MP4 / MOV）
2. 选择分辨率（720p / 1080p / 2K / 4K）
3. 点击「导出」
4. 系统处理完成后提供下载链接

**发布流程：**
1. 授权目标平台（抖音 / B站 / YouTube 等）
2. 填写发布信息：
   - 标题
   - 描述
   - 标签
   - 封面图
3. 选择立即发布或定时发布
4. 点击「发布」
5. 系统追踪发布状态并展示结果

## 六、非功能性需求

### 6.1 性能需求
- 页面加载时间 < 2 秒
- API 响应时间 < 500ms（不包含 AI 调用）
- 支持 1000+ 并发用户
- 视频生成任务排队时间 < 5 分钟（高峰期）

### 6.2 安全需求
- 用户密码加密存储（bcrypt）
- API 接口身份认证（JWT）
- 敏感数据传输加密（HTTPS）
- 定期数据备份
- 操作日志审计

### 6.3 可用性需求
- 系统可用性 > 99.5%
- 数据持久化存储
- 故障自动恢复
- 灰度发布支持

### 6.4 兼容性需求
- 支持主流浏览器（Chrome、Firefox、Safari、Edge 最新版本）
- 响应式设计，支持平板访问
- 移动端适配（小程序 / App 后续迭代）
