## 三、服务设计

### 3.1 API Gateway (cmd/api-gateway)
- **框架**: CloudWeGo Hertz
- **职责**:
  - 统一入口
  - 路由分发
  - 认证鉴权
  - 限流熔断
  - 请求日志

### 3.2 剧本生成服务 (cmd/script-service)
- **框架**: CloudWeGo Kitex
- **职责**:
  - 接收灵感输入
  - 调用文本模型生成剧本
  - 剧本结构化存储

### 3.3 分镜拆解服务 (cmd/storyboard-service)
- **框架**: CloudWeGo Kitex
- **职责**:
  - 解析剧本
  - 拆分镜头
  - 生成每个镜头的提示词

### 3.4 画面绘制服务 (cmd/image-service)
- **框架**: CloudWeGo Kitex
- **职责**:
  - 接收画面提示词
  - 调用图片模型生成画面
  - 画面存储和管理

### 3.5 视频生成服务 (cmd/video-service)
- **框架**: CloudWeGo Kitex
- **职责**:
  - 接收序列画面
  - 调用视频模型生成视频
  - 视频存储和管理

### 3.6 视频拼接服务 (cmd/merge-service)
- **框架**: CloudWeGo Kitex
- **职责**:
  - 接收多个视频片段
  - 使用 FFmpeg 拼接
  - 添加转场效果

### 3.7 模型管理服务 (cmd/model-service)
- **框架**: CloudWeGo Kitex
- **职责**:
  - 模型配置管理
  - 模型调用封装
  - 支持：
    - 文本模型: DeepSeek、Gemini 3.1 pro
    - 图片模型: Banana 2
    - 视频模型: Seedance

## 四、数据模型

### 4.1 项目表 (projects)
```go
type Project struct {
    ID          uint64    `gorm:"primaryKey"`
    UserID      uint64    `gorm:"index"`
    Title       string
    Description string
    Status      string    // pending, processing, completed, failed
    CreatedAt   time.Time
    UpdatedAt   time.Time
}
```

### 4.2 剧本表 (scripts)
```go
type Script struct {
    ID          uint64    `gorm:"primaryKey"`
    ProjectID   uint64    `gorm:"index"`
    Content     string    // 剧本内容
    Model       string    // 使用的模型
    Status      string
    CreatedAt   time.Time
    UpdatedAt   time.Time
}
```
