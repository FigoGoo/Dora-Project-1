## 一、概述

### 1.1 测试目标

- 验证各模块代码的正确性
- 提高代码质量，减少潜在缺陷
- 确保代码重构时功能正常
- 作为代码文档，展示函数的预期行为

### 1.2 测试范围

| 模块 | 包路径 | 说明 |
|------|--------|------|
| 配置模块 | `internal/pkg/config` | 配置加载和解析 |
| 日志模块 | `internal/pkg/logger` | 日志初始化和使用 |
| 中间件 | `internal/pkg/middleware` | HTTP/RPC 中间件 |
| 数据模型 | `internal/dal/db` | 数据库模型定义 |
| 工具函数 | `internal/pkg/utils` | 通用工具函数（待创建） |
| 业务逻辑 | `internal/biz/**` | 各服务业务逻辑（待创建） |

### 1.3 测试覆盖率目标

| 包类型 | 覆盖率目标 | 说明 |
|--------|-----------|------|
| 工具包 (pkg) | ≥ 90% | 通用工具需高覆盖率 |
| 业务逻辑层 (biz) | ≥ 85% | 核心业务逻辑必须覆盖 |
| 数据访问层 (dal) | ≥ 75% | 包含事务、错误处理 |

| 总体 | ≥ 80% | 整体代码覆盖率 |

## 四、测试用例设计规范

### 4.1 测试函数命名

```go
// 格式: Test{结构体名/函数名}_{场景描述}
func TestGenerateScript_Success(t *testing.T) {}
func TestGenerateScript_EmptyInspiration(t *testing.T) {}
func TestGenerateScript_ModelUnavailable(t *testing.T) {}

// 表格驱动测试
func TestValidatePrompt_TableDriven(t *testing.T) {}
```

### 4.2 测试结构 - AAA 模式

每个测试函数应遵循 Arrange-Act-Assert 模式：

```go
func TestXxx(t *testing.T) {
    // Arrange - 准备测试数据和环境
    input := "..."
    expected := "..."

    // Act - 执行被测试的代码
    result := FunctionUnderTest(input)

    // Assert - 验证结果
    assert.Equal(t, expected, result)
}
```

### 4.3 使用 testify/suite

```go
// 测试套件定义
type ScriptServiceTestSuite struct {
    suite.Suite
    service *ScriptService
    mockDB  *gomock.Controller
}

// 每个测试运行前执行
func (s *ScriptServiceTestSuite) SetupTest() {
    s.mockDB = gomock.NewController(s.T())
    s.service = NewScriptService( /* 依赖 */ )
}

// 每个测试运行后执行
func (s *ScriptServiceTestSuite) TearDownTest() {
    s.mockDB.Finish()
}

// 测试用例
func (s *ScriptServiceTestSuite) TestGenerateScript_Success() {
    // Arrange
    req := &GenerateScriptReq{
        ProjectId:   "proj-123",
        Inspiration: "一个关于冒险的故事",
    }

    // Act
    resp, err := s.service.GenerateScript(context.Background(), req)

    // Assert
    s.NoError(err)
    s.NotNil(resp)
    s.Equal("completed", resp.Data.Status)
}

// 运行测试套件
func TestScriptServiceTestSuite(t *testing.T) {
    suite.Run(t, new(ScriptServiceTestSuite))
}
```

package config

import (
    "os"
    "testing"
    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/suite"
)

type ConfigTestSuite struct {
    suite.Suite
    tempDir string
}

func (s *ConfigTestSuite) SetupTest() {
    s.tempDir = s.T().TempDir()
}

func (s *ConfigTestSuite) TestLoadConfig_Success() {
    // 创建临时配置文件
    configYAML := `
app:
  name: dora-test
  mode: test
server:
  host: 0.0.0.0
  port: 8080
`
    configPath := s.tempDir + "/config.yaml"
    err := os.WriteFile(configPath, []byte(configYAML), 0644)
    s.Require().NoError(err)

    // 测试加载
    err = Init(configPath)
    s.NoError(err)

    cfg := Get()
    s.Equal("dora-test", cfg.App.Name)
    s.Equal("test", cfg.App.Mode)
    s.Equal(8080, cfg.Server.Port)
}

func (s *ConfigTestSuite) TestMySQLConfig_DSN() {
    cfg := MySQLConfig{
        Host:     "localhost",
        Port:     3306,
        User:     "root",
        Password: "secret",
        Database: "dora",
    }

    dsn := cfg.DSN()
    s.Contains(dsn, "root:secret@tcp(localhost:3306)/dora")
}

func TestConfigTestSuite(t *testing.T) {
    suite.Run(t, new(ConfigTestSuite))
}
```

package script

import (
    "context"
    "testing"
    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/mock"
    "github.com/stretchr/testify/suite"
    "go.uber.org/mock/gomock"
)

// Mock 接口定义 (使用 gomock)
//go:generate mockgen -source=service.go -destination=mock_service.go -package=script

type ScriptServiceTestSuite struct {
    suite.Suite
    ctrl         *gomock.Controller
    mockModelSvc *MockModelService
    mockRepo     *MockScriptRepository
    service      *ScriptService
}

func (s *ScriptServiceTestSuite) SetupTest() {
    s.ctrl = gomock.NewController(s.T())
    s.mockModelSvc = NewMockModelService(s.ctrl)
    s.mockRepo = NewMockScriptRepository(s.ctrl)
    s.service = NewScriptService(s.mockModelSvc, s.mockRepo)
}

func (s *ScriptServiceTestSuite) TearDownTest() {
    s.ctrl.Finish()
}

func (s *ScriptServiceTestSuite) TestGenerateScript_Success() {
    // Arrange
    ctx := context.Background()
    req := &GenerateScriptReq{
        ProjectId:   "proj-123",
        Inspiration: "一个关于 AI 的故事",
        Model:       "deepseek",
    }

    expectedContent := "这是一个精彩的剧本..."
    s.mockModelSvc.EXPECT().
        GenerateText(gomock.Any(), req.Inspiration, req.Model).
        Return(expectedContent, nil)

    s.mockRepo.EXPECT().
        Save(gomock.Any(), gomock.Any()).
        DoAndReturn(func(ctx context.Context, script *Script) error {
            script.ID = 1
            return nil
        })

    // Act
    resp, err := s.service.GenerateScript(ctx, req)

    // Assert
    s.NoError(err)
    s.NotNil(resp)
    s.Equal(uint64(1), resp.Data.Id)
    s.Equal(expectedContent, resp.Data.Content)
    s.Equal("completed", resp.Data.Status)
}

func (s *ScriptServiceTestSuite) TestGenerateScript_EmptyInspiration() {
    // Arrange
    ctx := context.Background()
    req := &GenerateScriptReq{
        ProjectId:   "proj-123",
        Inspiration: "",
    }

    // Act
    resp, err := s.service.GenerateScript(ctx, req)

    // Assert
    s.Error(err)
    s.Nil(resp)
    s.Contains(err.Error(), "灵感不能为空")
}

func TestScriptServiceTestSuite(t *testing.T) {
    suite.Run(t, new(ScriptServiceTestSuite))
}
```

### 6.1 数据库 Mock

使用 `go-sqlmock`:

```go
package dal

import (
    "testing"
    "github.com/DATA-DOG/go-sqlmock"
    "github.com/stretchr/testify/assert"
    "gorm.io/driver/mysql"
    "gorm.io/gorm"
)

func TestProjectRepository_Create(t *testing.T) {
    // 创建 sqlmock
    db, mock, err := sqlmock.New()
    assert.NoError(t, err)
    defer db.Close()

    // 初始化 GORM
    gormDB, err := gorm.Open(mysql.New(mysql.Config{
        Conn: db,
        SkipInitializeWithVersion: true,
    }), &gorm.Config{})
    assert.NoError(t, err)

    repo := NewProjectRepository(gormDB)

    // 设置期望
    mock.ExpectBegin()
    mock.ExpectExec("INSERT INTO `projects`").
        WithArgs("测试项目", "描述", 1, "pending", 60, sqlmock.AnyArg(), sqlmock.AnyArg(), nil).
        WillReturnResult(sqlmock.NewResult(1, 1))
    mock.ExpectCommit()

    // 执行测试
    project := &Project{
        Title:       "测试项目",
        Description: "描述",
        UserID:      1,
        Status:      "pending",
        Duration:    60,
    }
    err = repo.Create(context.Background(), project)

    // 验证
    assert.NoError(t, err)
    assert.NoError(t, mock.ExpectationsWereMet())
}
```

## 八、测试执行

### 8.1 运行测试

```bash
# 运行所有测试
go test ./...

# 运行特定包的测试
go test ./internal/pkg/config -v

# 运行特定测试函数
go test ./internal/pkg/config -run TestLoadConfig_Success -v

# 显示测试覆盖率
go test ./... -cover

# 生成覆盖率报告
go test ./... -coverprofile=coverage.out
go tool cover -html=coverage.out -o coverage.html

# 并行运行测试
go test ./... -parallel 4
```

	go test -v ./... -bench=. -benchmem

## 十、最佳实践

### 10.1 测试编写原则

1. **独立性**: 每个测试应该独立运行，不依赖其他测试
2. **速度**: 单元测试应该快速执行
3. **可读性**: 测试代码应该清晰易读
4. **可维护性**: 测试代码应该易于修改
5. **确定性**: 相同输入总是产生相同输出

### 10.2 避免的陷阱

- ❌ 不要在测试中使用真实的外部服务
- ❌ 不要让测试之间相互依赖
- ❌ 不要在测试中使用随机数
- ❌ 不要测试私有方法（测试公共行为）
- ❌ 不要过度 Mock，保持测试的真实性

### 10.3 推荐的做法

- ✅ 使用表格驱动测试处理多种输入组合
- ✅ 为测试提供清晰的错误消息
- ✅ 测试边界条件和错误路径
- ✅ 保持测试简洁，每个测试只验证一件事
- ✅ 定期重构测试代码
